Designing a system to autonomously generate, edit, and sequence multimodality media from a single pitch is a heavy lift, but absolutely achievable with the right orchestration harness. A single monolithic agent will hallucinate or lose the thread on a task this complex.

To achieve the level of control, dynamic prompting, and human-in-the-loop (HITL) interaction you are looking for, we need to structure this as a **Multi-Agent Orchestration System** backed by a robust state machine.

Here is a blueprint for the agent harness, structured in TypeScript, designed to run smoothly on Google Cloud.

---

### 1. The Orchestration Architecture

Instead of one agent doing everything, split the system into a **Director (Orchestrator)** and specialized **Subagents**.

- **The Director (Planner & Orchestrator):** Interacts with the user, maintains the global state, generates the overarching AV script/storyboard, requests user approval, and dispatches tasks to subagents.
- **The Semantic Interface Negotiator:** Acts as middleware. It takes the Director's high-level creative intent (e.g., "Make it a moody, sweeping shot") and translates it into the strict, optimal prompt schemas required by specific generation models.
- **The Subagents (Parallelizable execution):** \* _Art Director Agent:_ Handles image generation, character sheets, and style consistency.
- _Cinematography Agent:_ Handles image-to-video generation, 360 spins, and camera motion.
- _Post-Production Agent:_ Handles FFmpeg/cloud-based concatenation, audio syncing, and editing.

### 2. The Shared State (Context Management)

The harness needs a strict data structure passed between the user, the Director, and the subagents. This prevents context window degradation and keeps the generation grounded.

```typescript
interface ProductionState {
    pitch: string;
    globalStyle: string; // e.g., "Cyberpunk", "1950s Noir"
    status:
        | "planning"
        | "pending_approval"
        | "generating_assets"
        | "editing"
        | "complete";
    avScript: {
        sceneId: string;
        description: string;
        shotType: string; // e.g., "Extreme Close Up", "Wide Establishing"
        durationSecs: number; // e.g., 5, 10
        imageReferenceUrl?: string; // Generated by Art Director
        videoAssetUrl?: string; // Generated by Cinematography Agent
        audioAssetUrl?: string; // Generated by Sound Agent
    }[];
    heroProps: Record<string, string>; // Maps prop name to image reference URL
    characterSheets: Record<string, string>; // Maps character to image reference URL
}
```

### 3. Tool & Skill Definitions

Tools must be highly modular. By defining strict schemas, the Director knows exactly which tool to call based on the user's initial prompt.

**Planning Tools (Director):**

- `draft_av_script`: Takes a story pitch and outputs a structured scene-by-scene array.
- `request_human_approval`: Halts the workflow, surfacing the `avScript` and `heroProps` to the user for editing before spending compute on video generation.

**Asset Generation Tools (Subagents):**

- `generate_character_sheet`: Inputs a description and `globalStyle`, outputs a reference image URL.
- `generate_scene_image`: Inputs scene description, `shotType`, and reference images (characters/props), outputs the base image for the shot.
- `animate_shot`: Inputs a starting image, motion prompt, and duration. This is where the Semantic Interface Negotiator routes to specific models (e.g., Veo for cinematic realism, Luma for specific motion control, or Nano Banana for specialized style pipelines).
- `generate_360_spin`: Specialized tool taking a single product reference and generating a turntable video.
- `virtual_try_on`: Takes a subject image and a garment image, returning a composite image ready for animation.

**Post-Production Tools:**

- `concatenate_timeline`: Triggers a Cloud Run worker running FFmpeg or a similar media processing library to stitch the `videoAssetUrl` array together.

### 4. Dynamic System Instructions (Prompt Injectors)

Hardcoding system prompts is too brittle for varied media. Instead, build a context-aware prompt injector. The Director assesses the task and loads specific "Best Practices" into the subagent's system instructions at runtime.

- **Cinematography Context:** If generating a video, inject rules like: _"Always specify camera movement (e.g., pan, tilt, dolly). For sweeping landscapes, append 'drone shot, 24mm lens'. For intimate dialogue, append 'shallow depth of field, 85mm lens'."_
- **Continuity Context:** _"You must strictly adhere to the provided character sheet reference images. Do not deviate from the established clothing or facial structures."_
- **Model-Specific Nuances:** The Semantic Interface Negotiator should know that Model A prefers natural language ("A realistic video of..."), while Model B responds better to booru-style tags ("cinematic, high contrast, 4k, octane render").

### 5. The Workflow Loop

1. **User Input:** "Create a story about a cyber-dog."
2. **Plan:** Director creates the `avScript`.
3. **Approve:** Director pauses. User reviews the plan, tweaks a scene description, and clicks "Approve."
4. **Parallel Execution:** \* Art Director generates the cyber-dog character sheet.

- Once the sheet is ready, a `ParallelAgent` execution spins up: Scene 1, Scene 2, and Scene 3 images are generated simultaneously using the reference sheet.

5. **Animation:** Cinematography agent takes the scene images and animates them based on the `durationSecs` and `shotType`.
6. **Assembly:** The Post-Production agent grabs all URLs and stitches them into a final `.mp4`.
